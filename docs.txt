POS Product Sync API Documentation

This document describes the REST API provided by the Pos Product Sync WordPress plugin for syncing products between your POS system and WooCommerce.

Base URL
The API endpoint is registered under the WordPress REST API infrastructure.

POST /wp-json/pos/v1/product

Authentication
Every request must include an Authorization header with a specific Bearer token.

Header Format:
Authorization: Bearer {token}

Endpoint Details

All operations (Create, Update, Delete) use the exact same endpoint. The action to perform is determined by the eventType field in the JSON request body.

1. Create a Product

Creates a new WooCommerce product and syncs POS-specific metadata.

Request Body (JSON):

eventType (String) [Required] - Must be exactly "create"
Id (String) [Required] - The unique Product ID from the POS system
Name (String) - Name of the product
Price (Number) - The regular selling price
OldPrice (Number) - Optional old regular price (used if Price is empty)
Code (String) - The product SKU
Description (String) - Product description
CurrentStock (Number) - Total stock quantity. Used if WarehouseList is empty.
WarehouseList (Array) - List of locations. Ex: [{"CurrentStock": 10}, {"CurrentStock": 5}]. The system will sum the CurrentStock across all warehouses.
CategoryName (String) - Name of the category. Will be created if it doesn't exist.
BrandName (String) - Stored as a product tag. Will be created if it doesn't exist.
Type (String) - POS Product Type
ProductBarcode (String) - Barcode string
UnitName (String) - Storage/Measurement unit name (e.g., 'kg', 'pcs')
CostPrice (Number) - Cost price from the POS
ImagePath (String) - Public URL of the product image. Will be downloaded and set as the featured image.

Example Request:
{
  "eventType": "create",
  "Id": "POS-1001",
  "Name": "Sample T-Shirt",
  "Price": 25.99,
  "Code": "TSHIRT-001",
  "Description": "A cool cotton t-shirt",
  "CurrentStock": 50,
  "CategoryName": "Clothing",
  "BrandName": "AwesomeBrand",
  "ProductBarcode": "123456789012",
  "CostPrice": 10.00,
  "ImagePath": "https://example.com/images/tshirt.jpg"
}

Success Response (201 Created):
{
  "success": true,
  "message": "Product created successfully",
  "data": {
    "product_id": 123
  }
}

Error Responses:
- 422 Unprocessable Entity: "Product Id required"
- 409 Conflict: "Product already exists"
- 500 Internal Server Error: "Failed to create product"

2. Update a Product

Updates an existing WooCommerce product that is synced with the provided POS ID. 

Request Body (JSON):

eventType (String) [Required] - Must be exactly "update"
Id (String) [Required] - The unique Product ID from the POS system
Name (String) - New name of the product
Price (Number) - New selling price
OldPrice (Number) - New old regular price
Code (String) - New product SKU
Description (String) - New product description
CurrentStock / WarehouseList (Number / Array) - Updates total stock quantity
CategoryName (String) - New category for the product
BrandName (String) - New product tag (brand)
ProductBarcode (String) - New barcode string
UnitName (String) - New unit name
CostPrice (Number) - New cost price
ImagePath (String) - Important: A new URL will download and replace the existing featured image.

Note: Omitted fields will generally retain their existing values in WooCommerce.

Example Request:
{
  "eventType": "update",
  "Id": "POS-1001",
  "Price": 19.99,
  "CurrentStock": 35
}

Success Response (200 OK):
{
  "success": true,
  "message": "Product updated successfully",
  "data": {
    "product_id": 123
  }
}

Error Responses:
- 422 Unprocessable Entity: "Product Id required"
- 404 Not Found: "Product not found"
- 500 Internal Server Error: "Failed to load product" / "Failed to update product"

-----------------------------------------------------

3. Delete a Product

Permanently deletes a WooCommerce product connected to the specific POS ID. It bypasses the trash bin.

Request Body (JSON):

eventType (String) [Required] - Must be exactly "delete"
Id (String) [Required] - The unique Product ID from the POS system to delete.

Example Request:
{
  "eventType": "delete",
  "Id": "POS-1001"
}

Success Response (200 OK):
{
  "success": true,
  "message": "Product deleted successfully",
  "data": []
}

Error Responses:
- 422 Unprocessable Entity: "Product Id required"
- 404 Not Found: "Product not found"
- 500 Internal Server Error: "Failed to load product"
